## Goal

Fix two distinct V7.1 regressions surfaced by the Pt 12 m20 pilot:

- **Regression 1 — silent pipeline exit between phases.** Both V7.1 builds (`build/a1/my-morning-20260528-014807` claude-tools and `build/a1/my-morning-20260528-060820` codex-tools) exit the wrapper subprocess silently mid-pipeline without raising or emitting a terminal event. Codex reached `phase_done python_qg` then stream ended. `v7_build.py:1453` only raises `LinearPipelineError` when `terminal_verdict != "PASS"`; on PASS the MDX phase should run unconditionally per `v7_build.py:1464+`. It did not.
- **Regression 2 — `translate` activity schema field drift.** V7.1 writer (both claude and codex) emits `translate` activity items with `prompt` + `answer` fields instead of canonical `source` (per `scripts/yaml_activities.py:630`). MDX assembly hard-fails: `ValueError: Failed to parse activity 7 type='translate': 'source'`. Pt 10's baseline activities.yaml at `.worktrees/builds/a1-my-morning-20260527-185032/curriculum/l2-uk-en/a1/my-morning/activities.yaml` uses canonical fields, so PR #2377's `linear-write.md` trim dropped a schema directive.
