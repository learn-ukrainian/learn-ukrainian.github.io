## Fix patterns (when fixing in place)

### `js/functionality-from-untrusted-source`

CodeQL fires when a script tag loads from a URL that's not pinned to an integrity hash:
```html
<script src="https://cdn.example.com/some.js"></script>
```

**Right fix — Subresource Integrity:**
```html
<script src="https://cdn.example.com/some.js"
        integrity="sha384-..."
        crossorigin="anonymous"></script>
```

Compute the SRI hash:
```bash
curl -s https://cdn.example.com/some.js | openssl dgst -sha384 -binary | openssl base64 -A
```

### `js/xss-through-dom`

CodeQL fires on `element.innerHTML = userInput` or `document.write(userInput)`.

**Right fix:**
```javascript
// Before:
el.innerHTML = userInput;
// After:
el.textContent = userInput;
// Or, if HTML rendering is genuinely needed, sanitize via DOMPurify (already a project dep? check first).
```

### `js/unvalidated-dynamic-method-call`

CodeQL fires on `obj[userKey]()` where `userKey` is user-controlled — could call any method, including unsafe ones like `__proto__` accessors.

**Right fix — explicit allow-list:**
```javascript
const ALLOWED_METHODS = new Set(["foo", "bar", "baz"]);
if (!ALLOWED_METHODS.has(userKey)) {
    throw new Error("invalid method");
}
obj[userKey]();
```
