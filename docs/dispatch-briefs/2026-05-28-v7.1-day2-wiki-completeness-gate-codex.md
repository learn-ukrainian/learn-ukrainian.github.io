The renderer pivot reframes the LLM from "compose lesson" to "render wiki + cited RAG + ULP pattern". For that to work, two new audit surfaces must exist:

1. **Upstream wiki completeness gate** — block the build BEFORE the writer runs if the wiki for the module is too thin to be a lesson spine.
2. **Layered vocab allowlist** — hard-reject content lemmas in the rendered module that aren't justified by wiki/plan/cumulative/closed-class/etc.

The first surface stops bad wiki → bad lesson; the second stops the renderer from quietly inventing vocab the wiki didn't authorize.

### Piece 2 — NEW `scripts/audit/wiki_completeness_gate.py`

Upstream hard-reject for thin wiki sections. Runs BEFORE the writer phase. **Mirror the shape of `scripts/audit/wiki_coverage_gate.py`** (function signatures, JSON output structure, gate-runner integration).

**Threshold table (per the ADR)**:

| Check | A1/A2 | B1+ | Seminar |
|---|---|---|---|
| Methodology section | non-empty | non-empty | non-empty + explicit pedagogical sequence |
| Steps in `## Послідовність введення` | ≥5 | ≥5 | ≥7 |
| L2 errors in `## Типові помилки` table | ≥3 rows | ≥3 rows | ≥3 rows + ≥1 per major grammar category |
| Decolonization pairs | ≥1 (for L1-Russian-substitutable topics) | ≥2 | ≥3 + cultural framing |
| Vocab minimum (lemma count) | ≥20 | ≥50 | ≥100 |
| Textbook exercise formats | ≥3 with chunk_id citations | ≥3 | ≥5 |
| Distractor inventory (wrong-form examples in L2 errors + decolonization pairs combined) | ≥6 | ≥10 | ≥15 |
| Chunk citation resolution | spot-check 3 random via `mcp__sources__verify_quote` | spot-check 5 random | **all chunks** verify_quote-validated |
| External article URL resolution | n/a | n/a | **all URLs** HTTP-resolve |

For this PR, implement A1/A2/B1+ thresholds end-to-end. **Seminar-specific checks (all-chunks + URL HTTP-resolve + two-source rule) can be stubbed with `NotImplementedError` and a clear seminar-deferred comment** — they require additional infrastructure not in scope for V7.1 Day 2 (separate seminar ADR planned).

### Piece 3 — extend `scripts/audit/checks/learner_state.py`

Replace the current single-source allowlist (cumulative-learner-state only) with the 8-source layered allowlist. Per the ADR:

```python
allowed_lemmas = (
    wiki.vocabulary_minimum_lemmas        # from piece 1's new manifest field
    | plan.targets.new_vocabulary

    | bad_form_markers                     # inside <!-- bad -->...<!-- /bad -->
```

**Reject condition (codex r2 + cursor r1 confirmed)**:
- Hard-fail on **unsupported content lemmas after VESUM normalization** (lemmas not in the allowlist that are also pedagogically meaningful — nouns/verbs/adjectives).
- NOT a single-token hard-fail; allow band-tolerated misses for tokenization edge cases. Band threshold per `scripts/config.py:488,493` precedent (current `max_unsupported_uk_words` = 80 for m20).
- Function words don't fail individually but are counted in the band.
