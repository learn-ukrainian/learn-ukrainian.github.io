## What to build

A linter that scans `docs/session-state/*.md` for environmental references that don't exist on the user's machine. Catches the bug class that bit on 2026-05-07 — handoff doc said `~/.bash_secrets` but `GH_TOKEN` actually lives in `.envrc` (corrected via direct main commit `4df94d0234`).

## Acceptance criteria

1. **New script** `scripts/audit/lint_session_state.py`:
   - `argparse` CLI: `--file PATH` or `--all` (scan all `docs/session-state/*.md`)
   - Detection rule: extract any markdown reference to a path matching one of these patterns:
     - `~/.bash_secrets`
     - `~/.bashrc`, `~/.zshrc`, `~/.zprofile`, `~/.profile`
     - `.envrc`, `.env`, `.env.local`
     - `~/.codex/config.toml`
     - `~/.gemini/settings.json`
     - `~/.claude/settings.json`
   - For each reference, check whether the file exists on disk relative to the project root (for project-scoped paths) or `$HOME` (for user-scoped). Skip references inside fenced code blocks marked as illustrative (e.g. `# example`), but do flag references in prose.
   - Allowlist: known "expected on user machine but not committed" paths in a small YAML file `scripts/audit/known_user_paths.yaml`. Pre-seed with `~/.codex/config.toml`, `~/.gemini/settings.json`.
   - Output: per-violation report `path:line: referenced env file '<path>' does not exist`
   - Exit 0 if clean, 1 if violations found
   - Conforms to `cli-help-standard.md`
2. **Tests** in `tests/audit/test_lint_session_state.py`:
   - Fixture: handoff doc referencing `~/.bash_secrets` (which doesn't exist) → linter exits 1, names the file
   - Fixture: handoff doc referencing `.envrc` (which does exist for the test) → linter exits 0
   - Fixture: handoff doc referencing `~/.codex/config.toml` (allowlisted) → linter exits 0
   - Fixture: code-block with example `~/.notreal` → linter does NOT flag (in fenced block)
3. **Pre-commit hook** in `.pre-commit-config.yaml` for `docs/session-state/*.md` only.
4. `docs/SCRIPTS.md` entry.
5. `ruff check` clean.
6. `.venv/bin/pytest tests/audit/test_lint_session_state.py -x` passes.

## Why this matters

A handoff doc that references files that don't exist sends the next agent on a wild goose chase or — worse — silently fails (Codex's #1783 dispatch hit a 401 because GH_TOKEN was assumed to be in `~/.bash_secrets` per the doc but was actually in `.envrc`). Static check at handoff-write time catches this before commit.
