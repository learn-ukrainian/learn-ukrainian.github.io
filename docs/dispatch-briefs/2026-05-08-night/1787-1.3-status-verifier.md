## What to build

A small CLI subcommand `delegate.py status-or-fail TASKID` that exits non-zero if the named task is NOT actually running, plus a memory rule prohibiting orchestrator from reporting async-task state from memory.

**Why this matters**: 2026-05-08 — orchestrator claimed bakeoff was "Gemini mid-write" when it had already finished. Memory of state lags reality; reality must be queried.

## Acceptance criteria

1. **New CLI subcommand** in `scripts/delegate.py`: `delegate.py status-or-fail TASKID` with these semantics:
   - Exit 0 if task is currently `running` per the runtime state
   - Exit 1 with stderr message `"task {TASKID} is not running (status={STATUS}, age={AGE}s)"` if task is not running, not found, or stale
   - Exit 2 if Monitor API unreachable (so caller can distinguish "infra failure" from "task gone")
   - Quiet by default; `--verbose` prints structured status
2. **Update** `scripts/delegate.py` `--help` and existing `status` subcommand docs to mention the new helper.
3. **New memory rule** in `claude_extensions/memory/MEMORY.md` — add as **#0G** between #0H and #0I:
   ```
   ## #0G — NEVER REPORT ASYNC-TASK STATE FROM MEMORY (2026-05-08)
   Before saying "task X is running / X just finished / X is at step Y", ALWAYS query
   `delegate.py status-or-fail X` or Monitor API `/api/delegate/active`. Memory of state
   from 2 minutes ago is wrong by default. Established 2026-05-08 after orchestrator
   reported bakeoff "Gemini mid-write" when it had finished.
   ```
4. **Run** `npm run claude:deploy` to sync `claude_extensions/memory/MEMORY.md` → `.claude/memory/`.
5. **Tests** in `tests/test_delegate.py` (extend if exists, create if not):
   - Fixture: dispatch a 5-second `sleep 5 && echo done` task; while it's running, `status-or-fail` exits 0
   - Same task, after it completes; `status-or-fail` exits 1 with informative message
   - Unknown task ID: exits 1 with "task not found"
   - Mock Monitor API down: exits 2
6. `ruff check` clean.
7. `.venv/bin/pytest tests/test_delegate.py -x` passes.
8. `docs/SCRIPTS.md` entry update.
