## Goal

Implement EPIC #1865 item #3 (in-tool cost telemetry footer). Each Monitor API response and each agent-tool-result envelope should grow a small **context-window telemetry footer** so the model can make runtime decisions about subagent vs inline, handoff triggering, and pacing — *without* relying on the model's own 1.8×-off self-estimate.

Format (text — appears at the tail of relevant tool outputs):

```
[ctx: 187K (+22K this turn), tier: base, 13K to premium, turn: 47]
```

Fields:

- `ctx:` total tokens-in-context (best available estimate from a deterministic source — see below).
- `(+NK this turn)` delta from previous turn.
- `tier:` `base` (≤200K) or `premium` (>200K).
- `NK to premium` countdown when in base; `NK over premium` when above.
- `turn:` monotonic turn counter from the session (if available).

Footer is **opt-in** per endpoint/tool — controlled by a single `LEARN_UKRAINIAN_TELEMETRY_FOOTER=1` env var. When disabled, no footer appears anywhere. Default: ON for inline-Claude session; OFF for dispatched subprocesses (`AGENT_NO_TELEMETRY_FOOTER=1` already implied by dispatch).

   ```python
   def render_footer(*, tokens: int, prev_tokens: int | None, turn: int | None) -> str:
       """Render the [ctx: ...] footer line. Pure; no I/O."""
       delta = f" (+{(tokens - prev_tokens) // 1000}K this turn)" if prev_tokens is not None else ""
       tier = "premium" if tokens > 200_000 else "base"
       distance = (200_000 - tokens) // 1000 if tier == "base" else (tokens - 200_000) // 1000
       distance_str = f", {distance}K {'to premium' if tier == 'base' else 'over premium'}"
       turn_str = f", turn: {turn}" if turn is not None else ""
       return f"[ctx: {tokens // 1000}K{delta}, tier: {tier}{distance_str}{turn_str}]"
   ```
