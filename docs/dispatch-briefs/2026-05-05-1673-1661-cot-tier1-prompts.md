4. **Design the writer-side CoT block.** Per #1673 AC: before drafting any section, the writer must reason out:
   - **Word budget:** total target × section weight, with an explicit per-section number
   - **Plan-vocab terms:** which `required_terms` from the plan must appear in this section
   - **Register:** what register the section's content type implies (narrative vs. dialog vs. exercise vs. rubric)
   - **Teaching sequence:** what order the wiki packet's claims imply (chronological? simple→complex? topic→example?)

   Block goes BEFORE drafting, in a `<thinking>` or `**Plan section X reasoning**` template. Should be 4-6 lines per section, not pages.

5. **Design the writer-side Tier-1 verification block** (from #1661):
   - **Verify all example words via `mcp__sources__verify_words` BEFORE listing.** If a word fails, OMIT (do NOT substitute confabulated alternative). Mark gaps explicitly.
   - **Modern Ukrainian only** unless explicitly discussing historical/etymology. Default to post-2019 Pravopys forms; do NOT default to OCS / Russian-shadow.
   - **Source citation discipline:** if you cite a dictionary/style-guide/author, the citation MUST be groundable in MCP via the matching tool. If you cannot ground it, do NOT cite — say "modern Ukrainian standardized form" instead of inventing a source.
   - **Quote attributions:** never fuse two separate quotes into a single attributed line. Verify exact wording exists in literary corpus before citing.
   - **End-of-output gate:** internal review pass before returning — re-scan every example word against the verification you ran; re-scan every cited source for grounding; OMIT or rewrite anything unverifiable.

6. **Design the reviewer-side per-dim CoT block.** Per #1673 AC: before scoring, the reviewer must list 2-3 specific evidence quotes from the prose AND state how each maps to the rubric. Operationalize NRR rule #6.

7. **Design the reviewer-side Tier-1 audit blocks** (from #1661, for linguistic-correctness dims: russicism, naturalness, decolonization):
