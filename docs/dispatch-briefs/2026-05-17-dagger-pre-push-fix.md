# Dispatch: fix local Dagger so pre-push hook isn't dead weight (#2057)

## Why this matters

`MEMORY.md #M-7` mandates pytest-before-push-to-main. The chosen path is
`dagger call pytest --source=.` (canonical, GHA-equivalent). PR #2056
landed a pre-push hook that runs that exact command. The hook
gracefully exits 0 when Dagger fails — which is **what's happening every
push tonight**, including all of #2068, #2069, #2070 in this session
(all required `--no-verify`). Local pytest-replay-before-push goes dark
until this is fixed.

Read `gh issue view 2057` for the full diagnosis. Two distinct symptoms:

1. **Container venv creation silently no-ops.** `python -m venv .venv`
   exits 0 in the `python:3.12.8-slim-bookworm` container, but
   # venv symlinked into worktree by delegate.py
   `.venv/bin/python` doesn't exist when the next `withExec` tries to
   call it. Then everything downstream fails with
   # venv symlinked into worktree by delegate.py
   `[Errno 2] No such file or directory: '/work/.venv/bin/python'`.
2. **16-minute "parsing command line arguments".** Dagger Cloud upload
   is retrying serially against `api.dagger.cloud` returning 401 (no
   `DAGGER_CLOUD_TOKEN` configured). Each retry takes minutes.

## Files

- `.dagger/src/learn_ukrainian_ci/main.py` — line 178-211 is the venv +

- `.dagger/pyproject.toml` / `.dagger/uv.lock` — if you adopt `uv`,
  reflect it here.
- `.git/hooks/pre-push` (generated by pre-commit) — no direct edit,

- `.pre-commit-config.yaml` — verify the dagger-pytest hook spec

## What to do (verifiable steps)

1. **Worktree setup.** Per `claude_extensions/rules/delegate-must-use-worktree.md`:
   you were spawned with `--worktree` so you're already in
   `.worktrees/dispatch/codex/dagger-pre-push-fix-2026-05-17/`. Verify with
   `git rev-parse --show-toplevel` and `git branch --show-current` (cite the
   raw output). All edits stay inside that worktree.

2. **Diagnose the venv silent-fail.** Reproduce locally first:
   ```bash
   docker run --rm -it python:3.12.8-slim-bookworm bash -lc 'python --version; which python; python -m venv /tmp/v; ls /tmp/v/bin/'
   ```
   Quote the raw output. If `python -m venv` doesn't actually populate
   `bin/`, that confirms the silent-fail. Likely cause: `ensurepip`
   missing from slim image, or `python` symlink missing (recent debian
   slim variants only ship `python3`).

3. **Pick a fix.** Either:
   - **A.** Change `["python", "-m", "venv", ".venv"]` →
     `["python3", "-m", "venv", "--without-pip", ".venv"]` + an explicit
     `get-pip.py` bootstrap (matches what older CI configs do when
     slim's ensurepip is missing).
   - **B.** Replace the venv block with `uv venv .venv` (uv is already
     in `.dagger/uv.lock` per the directory listing). One subprocess
     call; uv handles pip seeding itself. Probably the cleanest path.

   Pick one. Don't ship both. Whichever you pick, paste the new
   `.dagger/src/learn_ukrainian_ci/main.py:178-211` block in the PR
   description so a reviewer can verify the substitution without
   diffing.

4. **Verify locally.** Run `time dagger call pytest --source=. --keyword
   test_citation_matcher` (small, fast subset). Both verify-tasks:
   - Step succeeds end-to-end (exits 0, "passed in" line visible).
   - Total wall-time is sane (under 5min cold, under 1min warm). If
     it's still spending 16min on "parsing command line arguments", the
     401-loop is the dominant cost — handle it in step 5.
   - Quote the raw `time` output.

5. **Disable Dagger Cloud upload** (the 401 stall). Either:
   - Pass a flag that silences telemetry (check
     `dagger call --help` for `--no-cloud` or similar). Or:
   - Document in `.dagger/README.md` (create if missing) that
     `DAGGER_CLOUD_TOKEN` is optional and unsetting it does NOT skip
     telemetry uploads — the 401s are still attempted. If there's no
     flag, file follow-up #TBD and proceed with whatever workaround
     works (e.g. `DAGGER_NO_NAG=1` if that env exists, or document the
     stall as known).

6. **Commit on the worktree branch.** Conventional message:
   ```
   fix(dagger): venv creation in container + telemetry stall (#2057)
   ```
   Body must include: the verify-step raw output (step 4), the chosen
   fix variant (A vs B and why), and the closing reference `Closes #2057`.

7. **Push + open PR.** `git push -u origin {branch}` then
   `gh pr create --title "..." --body "..."`. PR title mirrors the
   commit subject. Body lists the test plan checklist (your local
   `time dagger call ...` results count as one checked item).
