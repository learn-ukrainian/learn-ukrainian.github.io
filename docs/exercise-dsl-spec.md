# Exercise DSL Specification

**Version:** 1.0
**Issue:** #996
**Used in:** V6 pipeline Steps 5 (placeholders) and 5b (filled exercises)

## Overview

The Exercise DSL uses Markdown-compatible `:::` fenced blocks. Two stages:

1. **Placeholder** (written by content author in Step 5): describes WHAT exercise goes WHERE
2. **Filled** (generated in Step 5b): contains the actual exercise content

Both use `:::` syntax that degrades gracefully as raw text if not parsed.

---

## Stage 1: Exercise Placeholders

Written by the content author. Describes intent, not syntax.

```markdown
:::exercise-placeholder
type: multiple-choice
tests: identify iotated vowel letters
items: 4
vocabulary: Я, Ю, Є, Ї, А, О, У, И, І
correct: Я, Ю, Є, Ї
theme: vowel sounds lesson
:::
```

### Fields

| Field | Required | Description |
|-------|----------|-------------|
| `type` | Yes | Exercise type (see types below) |
| `tests` | Yes | What skill/knowledge this tests (human-readable) |
| `items` | Yes | Number of items (3-8) |
| `vocabulary` | Yes | Exact Ukrainian words to use |
| `correct` | Yes | Correct answer(s) |
| `theme` | No | Narrative context for the exercise |
| `after` | No | Which concept this follows (for placement) |

### Constraint
`items` must not exceed the plausible combinations of `vocabulary`. If 4 words are provided, don't ask for 10 items.

## Stage 2: Filled Exercises

Generated by Step 5b (deterministic code or focused LLM session).

### Type: quiz (alias: multiple-choice)

```markdown
:::exercise[quiz]
Які голосні літери є йотованими?
- [ ] А
- [x] Я
- [x] Ю
- [ ] О
- [x] Є
- [x] Ї
- [ ] И
:::
```

### Type: fill-in (alias: cloze)

```markdown
:::exercise[fill-in]
к{и}т — кит (whale)
к{і}т — кіт (cat)
б{и}к — бик (bull)
б{і}к — бік (side)
:::
```

The letter in `{}` is blanked out for the learner. They type the missing letter.

### Type: match-up (alias: match)

```markdown
:::exercise[match-up]
| Left | Right |
|------|-------|
| В | [в] as in van |
| Н | [н] as in nest |
| Р | rolled [р] |
| С | [с] as in sun |
:::
```

Left and right columns are shuffled for the learner.

### Type: true-false

```markdown
:::exercise[true-false]
- [true] Ї is unique to the Ukrainian alphabet
- [false] Ukrainian has 33 sounds
- [true] Ukrainian has 33 letters
- [false] Ь makes a sound
:::
```

### Type: read-and-answer

```markdown
:::exercise[read-and-answer]
> Це Київ. Київ — столиця України. Тут є метро, аптеки, кафе.

1. Що це? — {Київ}
2. Що є тут? — {метро, аптеки, кафе}
:::
```

### Type: group-sort

```markdown
:::exercise[group-sort]
groups:
  - name: Голосні
    items: [А, О, У, Е, И, І]
  - name: Приголосні
    items: [Б, В, Г, Д, К, М]
:::
```

---

## Deterministic generation (no LLM needed)

These types can be generated by code from the placeholder fields:

| Type | How |
|------|-----|
| `match` | Split vocabulary by `=` separator, shuffle columns |
| `multiple-choice` (exact answers) | Use `correct` field for checked items, `vocabulary - correct` for unchecked |
| `cloze` (single letter) | Parse `correct` field for letter-swap pairs |
| `true-false` | Requires LLM (statements need context) |
| `read-and-answer` | Requires LLM (questions need comprehension) |
| `group-sort` | Parse `vocabulary` with group labels |

## DSL → MDX Conversion (Step 9)

Each DSL block maps to a React component:

| DSL (Gemini name) | Alias | MDX Component |
|--------------------|-------|---------------|
| `:::exercise[quiz]` | `multiple-choice` | `<Quiz>` |
| `:::exercise[fill-in]` | `cloze` | `<FillIn>` |
| `:::exercise[match-up]` | `match` | `<MatchUp>` |
| `:::exercise[true-false]` | — | `<TrueFalse>` |
| `:::exercise[read-and-answer]` | — | `<ReadingActivity>` |
| `:::exercise[group-sort]` | — | `<GroupSort>` |

The converter is deterministic Python code — no LLM needed.
