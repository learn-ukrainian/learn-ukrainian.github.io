## Known broken / workarounds

### Full v6 pipeline on Ukrainian input
- `.venv/bin/python scripts/build/v6_build.py a1 1 --step all` with `V6_WRITER_TEMPLATE=v6-write-uk.md` does NOT regenerate chunks. The chunked-write path reuses stale English chunks without invalidating on template change. Diagnosed by Codex `pipeline-cache-and-ratelimit` task; fix is #1381.
- Workaround: `rm -rf curriculum/l2-uk-en/a1/orchestration/<slug>` before each run. Works but defeats caching entirely.

### agent_runtime rate-limit throttle
- `scripts/agent_runtime/usage.py` defaults to 2 events / 5 min / 90s recency block. Self-wedges because blocked pre-call checks also count as events.
- Workaround: `LU_BYPASS_RATE_LIMIT=1` env var. Ladder (#1382 closed) handles the real fallback now.

### `GEMINI_API_KEY` in shell env
- Still set. Until user unsets it, Gemini CLI defaults to API mode (slower, uses $100/mo budget). Defensive env-strip in fallback.py handles this at runtime for OAuth rungs.
- Previous A1+A2 wikis were built via API mode (not Ultra subscription) because of this env var. Batch completed over multiple runs thanks to idempotent skip; user probably burned meaningful API dollars.
