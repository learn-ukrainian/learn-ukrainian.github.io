Auto-deploy hook — closing the manual-rsync gap

User friction trigger: "you have to deploy from claude_extensions ????"

The dual-tree design is real: claude_extensions/ is the committed source, .claude/ (plus .codex/, .gemini/, .agents/) is the deploy target. scripts/deploy_prompts.sh is the rsync. Without a hook, every edit needed manual npm run claude:deploy before it took effect — and forgetting it was silent staleness.

Shipped a FileChanged hook in claude_extensions/settings.json:

{
  "matcher": "claude_extensions/**",
  "hooks": [
    {
      "type": "command",
      "command": "$CLAUDE_PROJECT_DIR/.claude/hooks/auto-deploy-claude-extensions.sh",
      "timeout": 30
    }
  ]
}

The hook script (auto-deploy-claude-extensions.sh) runs npm run claude:deploy --silent. Idempotent rsync makes repeated fires safe. Failures surface via additionalContext; clean deploys stay silent to keep the agent transcript noise-free.

Bootstrap caveat: the hook itself needed one final manual deploy to land in .claude/hooks/. After that commit (70089b4e4d), future claude_extensions/ edits auto-propagate.
