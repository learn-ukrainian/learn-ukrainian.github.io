The #1900 finding in depth

Root cause: scripts/agent_runtime/adapters/codex.py::_rollout_matches_plan (lines 666-717) uses fail-fast return-on-first-match logic. Codex CLI 0.130.0 emits an AGENTS.md + <environment_context> envelope as a user-role message before the user's actual prompt. The matcher trips on this envelope, sees text ≠ plan.stdin_payload, returns False, and stops scanning. Never reaches the real prompt.

Chain reaction: _select_rollout_for_plan → None → _read_latest_rollout_trace → empty → parse_response returns tool_calls=[] → pipeline reports tool_calls_total=0. Has been happening since commit a5f96a2d28 on 2026-04-15.

Empirical evidence: codex's night-bakeoff rollout at ~/.codex/sessions/2026/05/12/rollout-2026-05-12T02-24-04-019e1991-c63e-7a33-8641-b8c35c36fbc7.jsonl has 8 successful response_item/function_call events + 8 successful event_msg/mcp_tool_call_end events. Codex was always working.

Fix design: change matcher from fail-fast to iterate-all-user-role-events. Mechanical, codex-dispatchable. In-flight.
