The two non-obvious diagnoses

Cold-start was dominated by static autoload, not the SessionStart hook

Six .claude/rules/*.md files load automatically into every Claude system prompt: critical-rules.md, non-negotiable-rules.md, workflow.md, delegate-must-use-worktree.md, cli-help-standard.md, model-assignment.md — 30.7 KB of always-load text. Plus four scoped rules that load when their paths get touched (which is "always" for active development). Total: 47.5 KB on every cold start.

/api/rules?format=markdown already served some of this content (22.3 KB, 3 of 6 unscoped files) but the Claude Code harness was reading the source files and the API was serving them in parallel. The fix: add the 3 missing files to RULE_SOURCES, drop a tiny stub into .claude/rules/_load-via-api.md pointing at the API, and skip the 6 unscoped files from the .claude/ rsync target only (Codex's .codex/rules/ and Gemini's .gemini/rules/ still get them — those agents' cold-start budgets aren't the Anthropic-quota pressure point).

act lost to Dagger after the data came in

First read (procedural): Dagger is a bigger upfront cost (pipeline rewrites in Python) than act (which replays existing YAML in 5 min setup). For the named pain (4m17s GHA loop), act looked like the right answer.

The data overruled. act's measured pytest cost was 7m45s — slower than GHA's 4m17s. Two structural reasons:
  • QEMU emulating amd64 on Apple Silicon adds ~2-3× CPU overhead for CPU-bound steps.
  • actions/cache@v4 is a no-op under act, so pip wheels (~600 MB torch CPU wheel included) re-download on every cold-container run.
Plus 7 environmental failures (4× rsync missing in the catthehacker image; 3× perf budgets violated under emulation).

Dagger trade differently: native aarch64 (no QEMU), explicit cache_volume mounts (pip survives across calls), pinned base image (you control what's installed — no missing rsync). First-run pytest 3m15s, warm-cache 2m23s. Faster than both GHA and act. 4 failures, three of which are missing npx/claude binary (fixable: add Node.js to the runner) and one borderline perf budget (15.30s vs 15s).

Lesson: the procedural pushback was correct given the cost difference; the data overruled. Both pushback and pivot were right at the time they happened. Net: act setup deleted in #1895; doc rewritten for Dagger.
