## Quick Reference Checklist

Before submitting a B2 checkpoint module, verify all items from `b2-module-template.md` PLUS:

### Checkpoint-Specific Requirements
- [ ] **TTT pedagogy:** Test-Teach-Test structure (comprehensive review)
- [ ] **Word count:** 1500+ words (lower than regular B2 modules' 1800+)
- [ ] **Vocabulary:** 40-50 items (review vocabulary from ALL modules in phase)
- [ ] **Activities:** 20+ comprehensive testing activities
- [ ] **Integration:** Complete review of ALL modules in the phase
- [ ] **Self-assessment:** "–ß–∏ –º–æ–∂–µ—Ç–µ –≤–∏..." checklist at end
- [ ] **Authentic texts:** 2-3 real Ukrainian texts (journalism, academic, official)
- [ ] **Emotional arc:** Uncertainty ("–ß–∏ –≥–æ—Ç–æ–≤—ñ –≤–∏?") ‚Üí Confidence ("–í–∏ –≥–æ—Ç–æ–≤—ñ!")
- [ ] **No new content:** Checkpoints ONLY review and test existing knowledge

## Checkpoint-Specific Activities

### Activity Format Quick Reference

**CRITICAL:** Use these exact formats for MDX generation to work correctly.

| Activity | Format |
|----------|--------|
| **quiz** | `- [ ] wrong` / `- [x] correct` with optional `> explanation` |
| **true-false** | `- [x] True.` with `> explanation` / `- [ ] False.` with `> explanation` |
| **fill-in** | `> [!answer] correct` + `> [!options] a \| b \| c \| d` |
| **error-correction** | ALL 4 required: `> [!error]` + `> [!answer]` + `> [!options]` + `> [!explanation]` |
| **match-up** | Table: `\| Left \| Right \|` |
| **group-sort** | `### Category` headers with `- items` underneath |
| **unjumble** | `> [!answer] Correct sentence here.` |
| **cloze** | Inline: `{blank\|opt1\|opt2\|answer}` |
| **select** | Multiple `- [x]` for all correct options |
| **translate** | Multi-choice: `- [x] Correct translation.` with `> explanation` |
| **mark-the-words** | `*marked*` words in blockquote passage |

### Comprehensive Testing (20+ activities required)

Checkpoints must test ALL content from the phase comprehensively:

**Core Testing Activities:**
1. **quiz** (3-4 activities, 10-14 items each) ‚Äî One quiz per major concept
2. **match-up** (2-3 activities, 14+ items) ‚Äî Term-definition, concept-example
3. **fill-in** (2-3 activities, 12-14 items) ‚Äî Contextual application
4. **group-sort** (2 activities, 16-20 items) ‚Äî Categorization by module/concept

**Integration Activities:**
5. **error-correction** (2 activities, 10-14 items) ‚Äî Common mistakes across phase
6. **cloze** (1 activity, 18+ blanks) ‚Äî Extended passage integrating all concepts
7. **mark-the-words** (1 activity, 400+ word passage) ‚Äî Identification across text
8. **translate** (1 activity, 10+ items) ‚Äî Translation across registers
9. **select** (1 activity, 12+ items) ‚Äî Multi-answer questions

**Production Activities:**
10. **Essay/Text with Model Answer** (1 activity, 200+ words)

## Common Pitfalls to Avoid

### 1. **Introducing New Content**
- ‚ùå Problem: Teaching new grammar or vocabulary in checkpoint
- ‚úÖ Solution: Checkpoints ONLY review and test. All content must come from prior modules.

### 2. **Insufficient Module Coverage**
- ‚ùå Problem: Testing only 2-3 modules out of 10-module phase
- ‚úÖ Solution: Create dedicated quiz for EACH major concept + integration quiz

### 3. **Missing Register Testing**
- ‚ùå Problem: Not testing register awareness
- ‚úÖ Solution: Include register classification activities, register transformation tasks

### 4. **No Model Answers**
- ‚ùå Problem: Writing tasks without examples
- ‚úÖ Solution: Every writing task includes complete model answer

### 5. **Too Few Activities**
- ‚ùå Problem: 14 activities (same as regular modules)
- ‚úÖ Solution: 20+ activities for comprehensive testing

### 6. **English in Content**
- ‚ùå Problem: English explanations or instructions
- ‚úÖ Solution: 100% Ukrainian immersion (English only in vocabulary translations)
