## Quick Reference Checklist

Before submitting a C2 checkpoint module, verify all items from `c2-module-template.md` PLUS:

### Checkpoint-Specific Requirements
- [ ] **TTT pedagogy:** Test-Teach-Test structure (comprehensive review)
- [ ] **Word count:** 2000+ words (review content, not new teaching)
- [ ] **Vocabulary:** 50-60 items (review vocabulary from ALL modules in phase)
- [ ] **Activities:** 25+ comprehensive testing activities
- [ ] **Integration:** Complete review of ALL modules in the phase
- [ ] **Self-assessment:** "–ß–∏ –º–æ–∂–µ—Ç–µ –≤–∏..." checklist at end

## Checkpoint-Specific Activities

### Comprehensive Testing (25+ activities required)

**Core Testing Activities:**
1. **quiz** (5-6 activities, 12-16 items each) ‚Äî One per major topic + integration
2. **match-up** (2-3 activities, 16+ items) ‚Äî Terminology from entire phase
3. **fill-in** (2-3 activities, 14-16 items) ‚Äî Nuanced contextual application
4. **group-sort** (2 activities, 20+ items) ‚Äî Categorization by module/concept

**Production Activities:**
5. **production** (2-3 activities) ‚Äî Creative/professional writing with Model Answers
6. **transform** (2 activities, 10+ items) ‚Äî Style/register transformation
7. **translate** (1-2 activities) ‚Äî Literary or professional translation

**Analysis Activities:**
8. **cloze** (2 activities, 22+ blanks) ‚Äî Extended passages integrating all concepts
9. **error-correction** (2 activities, 12-16 items) ‚Äî Stylistic/register errors
10. **mark-the-words** (1 activity, 600+ word passage) ‚Äî Identification across text

**Integration Activities:**
11. **text-comparison** (1 activity) ‚Äî Compare two texts from phase content
12. **style-analysis** (1 activity) ‚Äî Identify all stylistic devices in passage

## Common Pitfalls to Avoid

### 1. **Introducing New Content**
- ‚ùå Problem: Teaching new concepts in checkpoint
- ‚úÖ Solution: Checkpoints ONLY review and test. All content must come from prior modules.

### 2. **Insufficient Phase Coverage**
- ‚ùå Problem: Testing only 2-3 topics out of 10-20 covered in phase
- ‚úÖ Solution: Create dedicated quiz for EACH major topic + integration quiz

### 3. **Missing Production Assessment**
- ‚ùå Problem: Only multiple choice, no creative/professional production
- ‚úÖ Solution: Include substantial production tasks with Model Answers and rubrics

### 4. **Shallow Self-Assessment**
- ‚ùå Problem: Generic "–ß–∏ –º–æ–∂–µ—Ç–µ –≤–∏..." questions
- ‚úÖ Solution: Specific criteria tied to each module cluster in phase

### 5. **Too Few Activities**
- ‚ùå Problem: 14-16 activities (same as regular C2 modules)
- ‚úÖ Solution: 25+ activities for comprehensive testing

### 6. **Missing Model Answers**
- ‚ùå Problem: Writing tasks without examples
- ‚úÖ Solution: ALL production tasks include complete Model Answers

### 7. **No Integration Testing**
- ‚ùå Problem: Testing topics in isolation
- ‚úÖ Solution: Dedicated integration section combining all phase concepts
