Проте на рівні 7–8 класів підручники використовують цей домен для пояснення складніших морфологічних і синтаксичних структур, які є критично важливими для комунікації на рівні B2: безособових дієслівних форм на *-но, -то* [S2] та синтаксису прикладок (назв компаній, програм, систем) [S3].

Підручники з української мови для 7-го класу запроваджують форми на *-но, -то* безпосередньо в контексті технологій, пояснюючи їхню функцію так: «Основне призначення конструкцій із формами на -но, -то – зосереджувати увагу не на виконавцеві, а на самій дії, на її завершенні, результаті. Такі конструкції допомагають стисло й точно висловити думку» [S2]. Для школярів цей перехід ілюструють прямою трансформацією з пасивного дієприкметника або двоскладного речення: *прибрана кімната – кімнату прибрано*; *депутати прийняли закон – закон прийнято* [S2]. У контексті технологій це пояснюється на прикладі роботи систем: *«На основі штучного інтелекту (ШІ) створено багато інструментів»* [S2].

У 8-му класі лексика сфери інформаційних технологій (корпорації, мережі, роботи) стає ідеальним полігоном для вивчення прикладок. Підручники пропонують тексти-роздуми про вплив ШІ на людство, де учні аналізують граматичне узгодження родових понять (мережа, корпорація, робот) та власних назв («Фейсбук», «Майкрософт», Софія) [S3]. Також на цьому етапі активно повторюються неозначено-особові речення (модель *Відремонтували автобусну зупинку → Відремонтували* [S3]), які в українській мові часто виконують ту саму функцію, що й англійський пасив, коли виконавець невідомий або неважливий.

Для іншомовних учнів (L2) цей домен є ідеальним для засвоєння української безособовості. Англійська мова в ІТ-сфері тяжіє до пасивного стану (The file was downloaded, The system has been updated). Українська мова, натомість, оперує формами на *-но, -то* (Файл завантажено) або активними / неозначено-особовими конструкціями (Програма оновила систему / Систему оновили).

## Повна парадигма

### 1. Творення безособових форм на -но, -то

Ці незмінні форми є синтаксичним ядром для опису роботи комп'ютерних систем, інтерфейсів та алгоритмів. Вони вказують на результативність і доконаність дії.

**Загальна формула творення (ланцюжок перетворення):**
`{інфінітив} → {основа пасивного дієприкметника} → {+но / +то} → {кінцева форма}`

| Група дієслів | Інфінітив | Дієприкметник | Трансформація | Кінцева форма на -но/-то |
| :--- | :--- | :--- | :--- | :--- |
| **Дієслова на -ти (основа на голосний)** | створити | створений | створе- → +но | **створено** [S2] |
| | розробити | розроблений | розробле- → +но | **розроблено** [S3] |
| | зберегти | збережений | збереже- → +но | **збережено** |
| | завантажити | завантажений | завантаже- → +но | **завантажено** |
| | надіслати | надісланий | надісла- → +но | **надіслано** |
| | прибрати | прибраний | прибра- → +но | **прибрано** [S2] |
| **Дієслова з суфіксом -ну- / односкладові** | прийняти | прийнятий | прийня- → +то | **прийнято** [S2] |
| | відкрити | відкритий | відкри- → +то | **відкрито** |
| | стерти | стертий | стер- → +то | **стерто** |
| | натиснути | натиснутий | натисну- → +то | **натиснуто** |

**Критичні граматичні та синтаксичні обмеження:**

1. **Транзитивність:** Форми на *-но, -то* утворюються ВИКЛЮЧНО від перехідних дієслів (дія яких може переходити на прямий об'єкт) `<!-- VERIFY: СУМ, академічна граматика -->`. Неможливо утворити таку форму від дієслова *працювати* (немає форми *\*працьовано*).
2. **Відмінок об'єкта:** Іменник, на який спрямована дія, завжди стоїть у **знахідному відмінку (Accusative) без прийменника** [S2]. Він є прямим додатком, а не підметом. Наприклад: *кімнат-у (Зн.в.) прибрано*; *закон-Ø (Зн.в.) прийнято* [S2].
3. **Відсутність підмета:** Це безособові односкладні речення. Вони принципово не можуть мати підмета в називному відмінку.
4. **Відсутність агенса в орудному відмінку:** За нормами класичної української стилістики, при формах на *-но, -то* не вказується виконавець дії в орудному відмінку (не можна сказати *\*програму створено розробником*). Якщо виконавця треба назвати, використовується активне речення (*розробник створив програму*) `<!-- VERIFY: Стилістика Антоненка-Давидовича -->`. Якщо ж інструмент є неживим засобом, можна використати конструкцію з прийменником: *створено за допомогою штучного інтелекту* [S1].

### 2. Синтаксис прикладок (Appositions) в ІТ-сфері

Технологічний домен насичений назвами компаній, платформ та застосунків. Їх узгодження викликає багато труднощів у L2.

| Тип прикладки | Родове поняття (відмінюється) | Власна назва (не відмінюється, у лапках) | Приклад у тексті |
| :--- | :--- | :--- | :--- |
| **Корпорації** | голова корпорації (Род.в.) | «Майкрософт» (Наз.в.) | голова корпорації «Майкрософт» [S3] |
| **Мережі** | засновник мережі (Род.в.) | «Фейсбук» (Наз.в.) | засновник мережі «Фейсбук» [S3] |

**Ланцюжок правила:** `{Родове поняття у потрібному відмінку} + {Власна назва у лапках ЗАВЖДИ в називному відмінку}`.

**Винятки (без лапок):**
Якщо власна назва не є умовною маркою, а є унікальним ім'ям істоти (навіть штучної) або астрономічним об'єктом, лапки не ставляться, а ім'я відмінюється:
- *робот Софія* [S3] → *робота Софії*
- *планета Юпітер* [S3] → *на планеті Юпітер*

Загальноприйняті терміни, що втратили статус власної назви, пишуться з малої літери й відмінюються самостійно без родового слова (хоча в офіційних або старіших текстах можливе написання з великої та в лапках): *винайдення інтернету* [S5] (сучасна норма) порівняно зі старішим/офіційним *у мережі «Інтернет»* [S1].

## Частотність і пріоритети

На рівні B2 форми на *-но, -то* є **найвищим пріоритетом** для рецептивного та продуктивного засвоєння в контексті роботи з інформацією. В україномовних інтерфейсах операційних систем (Windows, macOS, iOS) та застосунків саме ці форми забезпечують зворотний зв'язок із користувачем: *повідомлення надіслано, пристрій підключено, пароль змінено, помилку виявлено*. Без глибокого розуміння цієї структури учень B2 не зможе вільно орієнтуватися в українському цифровому просторі.

Неозначено-особові речення (модель *Відремонтували автобусну зупинку* [S3]) посідають друге місце за пріоритетністю. Вони частіше вживаються в розмовному стилі та новинах, коли йдеться про дії групи людей (розробників, хакерів, інженерів): *систему зламали, застосунок оновили*.

Вміння правильно узгоджувати прикладки (модель *у мережі «Фейсбук»*, а не *\*у мережі Фейсбуці*) є критичним для академічного та професійного письма на етапі переходу до рівня C1.

## Деколонізаційні застереження

Найпотужнішим наслідком російського мовного впливу в технологічній сфері є зловживання зворотними дієсловами на *-ся* у пасивному значенні. Російська мова масово генерує конструкції на кшталт *«файл скачивается», «программа устанавливается», «данные обрабатываются»*. 

В українській мові частка *-ся* (історично — займенник *себе*) зберігає потужне рефлексивне або самочинне значення. Коли ми кажемо *«двері зачиняються»*, це означає, що вони роблять це самі, автоматично. Але коли йдеться про дію, ініційовану користувачем або прихованим агентом (сервером), використання *-ся* є неприродним калькуванням. Українська мова є мовою активної дії. 

Замість пасивних потвор на *-ся*, слід завжди віддавати перевагу:
1. **Активним конструкціям:** *Програма завантажує файл* (замість *файл завантажується*).
2. **Неозначено-особовим конструкціям:** *Файл завантажують* (хтось невідомий це робить).
3. **Результативним формам на -но, -то:** *Файл завантажено* (коли дія завершена).

Письменник модуля повинен суворо уникати перекладу англійського пасиву через українське *-ся*, крім випадків щирої самочинності механізмів.

## Природні приклади

Приклади згруповано за синтаксичними моделями, що відображають ключові патерни опису технологій. Усі речення базуються на матеріалах українських шкільних підручників.

**Група 1: Безособові конструкції результату дії (форми на -но, -то)**
Ці приклади ілюструють фокус на завершенні дії без вказівки на виконавця. Зверніть увагу на знахідний відмінок об'єкта.
* На основі штучного інтелекту (ШІ) створено багато інструментів. [S2]
* Депутати прийняли закон – закон прийнято. [S2]
* Прибрана кімната – кімнату прибрано. [S2]

**Група 2: Власні назви в ІТ (Прикладки та узгодження)**
Ілюстрація незмінності брендових назв у лапках та відмінювання імен штучних інтелектів (роботів).
* Фізик Стівен Гокінг і голова корпорації «Майкрософт» Білл Гейтс неодноразово застерігали від розроблення штучного інтелекту. [S3]
* Засновник мережі «Фейсбук» Марк Цукерберг переконаний, що штучний інтелект допоможе впоратися з багатьма проблемами. [S3]
* Якщо не говорити про апокаліпсис, то в людства з’являються величезні можливості, – переконаний керівник Центру робототехніки «Ботеон» Олексій Резніченко. [S3]
* Водночас робот Софія навчається, жартує і дає інтерв’ю. [S3]
* Планета Юпітер не має твердої поверхні й складається із газу. [S3]

**Група 3: Неозначено-особові та активні дії технологій**
Коли ми акцентуємо увагу на тому, що систему як інструмент хтось використовує, або система сама стає активним суб'єктом.
* Відремонтували автобусну зупинку. → Відремонтували. [S3]
* Технології вже сьогодні допомагають розв’язувати проблеми забруднення довкілля. [S3]
* У мережі «Інтернет» існують сервіси, що дозволяють за допомогою штучного інтелекту створювати тексти й зображення. [S1]

**Група 4: Синтаксис цифрового спілкування та мереж**
Специфічна еліптична (скорочена) мова чатів та обговорення мереж.
* Привіт! Ти прийдеш сьогодні на тренування? Тобто? Ти поспішаєш? Не будеш? [S4]
* Соціальні мережі — це не альтернатива дружбі, це її продовження. [S4]
* Одним із найпоширеніших винаходів сучасності стали соціальні мережі. [S5]

## Рекомендації для вправ

Під час створення вправ для рівня B2 автор повинен провести учня від розпізнавання інтерфейсних повідомлень до самостійного продукування безособових конструкцій.

**Фаза 1: Рецептивна ідентифікація та аналіз**
Учням пропонується скріншот україномовного інтерфейсу (мобільний телефон, соціальна мережа) або короткий текст про ШІ.
* **Завдання:** Знайти всі форми на *-но, -то* (*збережено, відкрито, заблоковано*).
* **Аналітичний крок:** Визначити об'єкт дії та переконатися, що він стоїть у знахідному відмінку (Accusative). Учні мають усвідомити: підмета тут немає.

**Фаза 2: Механічна трансформація (Злам патерну)**
Оскільки англомовні учні думають категоріями пасивного стану, їх треба навчити перемикати цей регістр.
* **Завдання на усунення "was/were":** Дано неправильні речення L2 (*Файл був збережений*). Учень має утворити правильну нормативну форму (*Файл збережено*).
* **Трансформація активу в результат:** Зразок підручника 7 класу [S2]: *Користувач змінив пароль* → *Пароль змінено*. *Розробники створили інструмент* [S2] → *Інструмент створено*.

**Фаза 3: Продукція в контексті ІТ**
Учні виконують роль системного адміністратора або програміста, який пише звіт про виконану роботу або створює спливаючі повідомлення для нового застосунку.
* **Завдання:** Використовуючи інфінітиви (*скопіювати, надіслати, видалити, завантажити*), написати системні статуси для 5 різних операцій, не забуваючи правильно ставити іменник у знахідний відмінок (*Дані скопійовано*, *Фотографію видалено*).
* **Робота з прикладками:** Вставити правильні форми власних назв ІТ-компаній у текст, вирішуючи, де ставити лапки і який відмінок використовувати (напр., *Він працює в корпорації «Майкрософт»* [S3]).

## Зв'язки з іншими темами

**Передумови:**
Для успішного засвоєння цієї теми учень вже повинен впевнено володіти Знахідним відмінком (Accusative case) для неістот та істот, оскільки форми на *-но, -то* вимагають прямого додатка [S2]. Також необхідне розуміння концепції доконаного виду дієслова (Perfective aspect), адже безособові форми походять переважно від дієслів, що означають завершену дію (зробити → зроблено, але не робити → роблено). Учень має знати парадигму пасивних дієприкметників минулого часу, яка є морфологічною базою для цих утворень.

**Що ця тема уможливлює:**
Опанування безособових результативних конструкцій та прикладок відкриває двері до читання складних технічних текстів, розуміння інструкцій, умов використання (Terms of Service), а також вільної навігації україномовним цифровим простором. Це прямий крок до професійного мовлення рівня C1 (написання формальних звітів, технічної документації, ділового листування).
