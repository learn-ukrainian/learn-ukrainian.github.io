# 0-10 Scoring System for Content Quality Reviews

**CRITICAL INSTRUCTION:** Use this 0-10 scale for ALL dimension scores. High scores require justification. Perfect scores (10/10) should be rare.

---

## Scoring Philosophy

**Score Distribution:**
- **0-4 = FAIL** ‚Üí Auto-fail gates, critical issues, must fix before passing
- **5-6 = PASS (Minimum)** ‚Üí Meets gates, acceptable quality, has noticeable weaknesses
- **7-8 = GOOD** ‚Üí Solid quality, minor improvements possible, above minimum standards
- **9-10 = EXCELLENT** ‚Üí Exceptional quality, reference-worthy, innovative or exemplary

**Mandatory Justification Rule:**
- **For scores ‚â•8:** Explain what would make it higher (or why it's already 10/10)
- **For scores ‚â§6:** Explain specific weaknesses and what's missing
- **For score 7:** Explain both: what's good AND what could be better

## Dimension Rubrics

## Overall Score Calculation

**Method 1: Average (Simple)**
```
Overall = (Sum of all 10 dimensions) / 10
```

**Method 2: Weighted (Recommended)**
```
Overall = (
    Coherence √ó 1.0 +
    Relevance √ó 1.0 +
    Educational √ó 1.2 +  // Extra weight: core learning value
    Language √ó 1.1 +     // Extra weight: Ukrainian quality critical
    Pedagogy √ó 1.2 +     // Extra weight: teaching approach critical
    Immersion √ó 0.8 +    // Lower weight: binary (right level or not)
    Activities √ó 1.3 +   // Extra weight: primary learning tool
    Richness √ó 0.9 +
    Humanity √ó 0.8 +
    LLM Fingerprint √ó 1.1  // Extra weight: AI detection critical for B1+
) / 10.4
```

**Rounding rule:** Round to nearest 0.5 (e.g., 7.3 ‚Üí 7.0, 7.6 ‚Üí 8.0, 7.5 ‚Üí 8.0)

## Testing the Scale

**To verify the scale is working:**

1. **Check score distribution** across 10-20 reviewed modules:
   - Expected: Bell curve centered at 7-8
   - Problem: All scores 9-10 (too generous) or all scores 5-6 (too harsh)

2. **Verify differentiation:**
   - A "good" module (7/10) should score clearly lower than "excellent" (9/10)
   - Justifications should explain the difference

3. **Check for score inflation:**
   - Are you giving 9-10 too easily?
   - Do justifications genuinely explain why score is high?

4. **Validate with human judgment:**
   - Does the score "feel right" when you read the module?
   - Would you recommend a 10/10 module as reference quality to other writers?

---

## Common Scoring Mistakes to Avoid

1. **Anchoring to 10/10** - Don't start assuming everything is perfect and deduct points. Start neutral and build up.

2. **Ignoring small issues** - "It's mostly good" ‚Üí 9/10. NO. If there are issues, score reflects them (7-8/10).

3. **Not using low scores** - If module has problems, give 5-6/10. That's what the scale is for.

4. **Missing justifications** - Every score ‚â•8 or ‚â§6 MUST have explanation of what's missing or what makes it exceptional.

5. **Inconsistent standards** - A 7/10 in Coherence should represent similar quality level as 7/10 in Activities.

6. **False equivalence** - "Meets all gates" ‚â† automatic 8/10. Gates are minimum (5-6/10). Good quality is 7-8/10. Exceptional is 9-10/10.

---

## Final Reminder

**High scores are earned, not given by default.**

- 10/10 = Exceptional, reference quality, other modules should aspire to this
- 9/10 = Excellent, very few weaknesses, above expectations
- 8/10 = Good, solid quality, minor improvements possible
- 7/10 = Adequate, meets standards, noticeable room for improvement
- 6/10 = Acceptable, passes gates, has several weaknesses
- 5/10 = Barely passing, meets minimum, significant weaknesses
- 0-4/10 = Fail

**If you find yourself giving mostly 9-10s, you're being too generous. Recalibrate.**
