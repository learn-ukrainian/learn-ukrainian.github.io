1. Executive summary

A one-session audit using 4 independent agents + a 7-model writer bakeoff + verified API pricing produced five concrete routing findings, one prompt-architecture finding, and three open questions. Everything is sourced; nothing is from category bins.

1

The matrix's $/$$/$$$ cost bins were wrong

Verified API pricing shows Codex GPT-5.5 output is $30/M — more expensive than Claude Opus 4.7 at $25/M. Gemini 3.1 Pro is NOT "0$ unmetered" — it's $2/$12 paid API. DeepSeek-v4-pro's 75% discount expires 2026-05-31 — 4× cost cliff incoming.

2

DeepSeek-Pro produces the best Ukrainian content, full stop

Despite dropping the prompt's visible-CoT contract, DeepSeek-Pro's m20 module output beat all 4 qwen variants on Ukrainian pedagogical quality, immersion-discipline, and pre-injected activity references. The "fail" was structural; the substance is gold.

3

Visible CoT in V7 writer is vestigial

Unanimous 3-agent consensus (Codex + Gemini + DeepSeek): visible chain-of-thought adds ~20% output overhead and is not parsed by any downstream gate. Removing it saves $0.005–$0.025/build + 30–90s wall time + unblocks DeepSeek-Pro for the writer role.

4

Family diversity beats variant diversity at low cost

The latest-of-family rule (your kubedojo critique) was correct: qwen-3.6-plus vs qwen-3.6-max-preview produced near-identical contract compliance and content quality at 3× cost difference. Pay for vendor diversity (Qwen vs DeepSeek vs GLM vs Kimi vs MiniMax), not for higher tiers within one vendor. v2 update: minimax-m2.7 (recovered after the Hermes patch) adds a non-Qwen non-Chinese-open-only data point — it fits the universal-failure pattern but produces competent Ukrainian content, broadening the cross-family confidence interval for the finding.

5

Adversarial review is the one role where CoT IS load-bearing

Unanimous: adversarial review = REQUIRED visible CoT. The reasoning IS the deliverable. Same for Q&A justification, structured ADR critique. Everywhere else (writer, content review, code review, search): REMOVE or OPTIONAL.

Bottom line
Your matrix structure is right; its numbers and the writer prompt's CoT requirement aren't. Three concrete changes will save real money and raise quality: (a) verified $/M pricing replaces categorical bins in matrix v1.3, (b) writer prompt drops visible-CoT (matches DeepSeek's empirical behavior), (c) deepseek-v4-pro promotes from content-reviewer-only to writer-candidate once CoT requirement is dropped — potentially saving $4–12K of full-curriculum writer spend at deepseek-pro post-discount rates.

2. Methodology

Principle: quality first, cost second

Per the project's #1 QUALITY ABOVE ALL non-negotiable rule: every verdict starts with a binary PASS / FAIL on the quality contract. Cost only comes into play among quality-passing models. A cheap model that fails contract isn't "almost there" — it's out of the running. Cost optimization waits.

Two parallel evaluation tracks

  1. Independent peer review of our existing routing matrix — four agents (Codex GPT-5.5, Gemini 3.1 Pro, DeepSeek v4 Pro, Qwen 3.6 Plus) were given an identical prompt blind to model identity, asked to produce a fixed-schema route table for 11 task types with anti-fabrication rules (every $/M cost must cite a pricing-page URL + date checked).
  2. Head-to-head writer bakeoff — seven candidate models received the actual m20 (a1/my-morning) writer prompt from a real V7 build (~60K tokens) and produced module output. Each output was graded on (a) writer-contract compliance (4 plan_reasoning blocks + audit lines + correct fences) and (b) content quality (Ukrainian pedagogy, immersion discipline, gate-passing markers).

The latest-only rule

For each model family, only the latest variant was tested. Initial picks (mistral-nemo at $0.02/M, arcee-trinity-thinking, kimi-k2.5) were corrected mid-session to the actual latest: mistral-large-2512, kimi-k2.6, glm-5.1. Older or smaller variants are not informative about the current capability of a family.

Anti-fabrication discipline (#M-4)

Every numerical claim in this report is backed by a tool call recorded during the session. Pricing comes from vendor pricing pages (URLs cited inline). Bakeoff verdicts come from grep + file-size measurements on the actual model outputs (linked in the appendix). Cross-agent quotes are verbatim from the conversation thread.

Conflict-of-interest handling

The 4-agent routing assessment was blind: agents were not told which model they were. Each was asked to disclose at the end if they suspected they were one of the named models. Codex correctly self-identified (rating itself for novel implementation while honestly flagging Ukrainian linguistic limits). Other agents did the same. The bakeoff was head-to-head on identical input, eliminating self-rating bias.

The two-axis verdict — content vs contract are separable

Plot twist (see §6 for the full deep-dive)
DeepSeek-v4-pro was the only model that dropped the visible-CoT contract entirely (0 plan_reasoning blocks, 0 audit lines) AND simultaneously produced the highest-quality Ukrainian content of any challenger. The qwen family did the opposite — full contract compliance but lower-quality content (immersion violations like English glosses inside uk= attributes). The bakeoff revealed that "contract pass" and "content quality" are independent axes, and a one-dimensional pass/fail grade hid the real story.

The two universal failures (all challengers)

  1. Wrong fence syntax — every challenger emitted ```markdown file=module.md instead of the pipeline-expected ```module.md. This is the GitHub/Common-Markdown default, embedded in their training data. Fixable with one explicit prompt directive.
  2. JSON in .yaml blocks — every challenger emitted JSON inside the activities.yaml / vocabulary.yaml / resources.yaml fences instead of YAML. The pipeline's YAML parser will reject. Fixable with one explicit prompt directive.

Both failures are prompt-clarity bugs, not model defects. Tightening these two directives unblocks every challenger structurally.

v2 update: The minimax-m2.7 verdict (recovered after the Hermes patch) confirms both universal failures on a non-Qwen non-Chinese-open-source-only data point. Pattern is now confirmed on 8 of 8 graded challengers (every model except the claude-tools baseline). The prompt-clarity directives' priority is reinforced; the CoT-removal dispatch fired this session addresses both alongside the scaffolding removal.

6. DeepSeek introspection — the plot twist

When deepseek-v4-pro's writer output dropped the entire <plan_reasoning> contract while producing the best content, we asked it directly to explain why. Its response — quoted verbatim below — is one of the most useful single artifacts produced in this session.

The four-question introspection

Q1: Why did you skip <plan_reasoning> blocks?

"The prompt's artifact fences (module.md, activities.yaml, vocabulary.yaml, resources.yaml) are heavyweight, concrete, and positioned as the terminal deliverable. They exert far more gravitational pull than internal CoT blocks. The <plan_reasoning> sections, even when flagged 'mandatory,' read as scaffolding — a means to the artifact, not the artifact itself. In a long prompt, the model compresses: scaffolding collapses, deliverables survive." — DeepSeek v4 Pro, self-introspection 2026-05-19

Q2: Same for the audit lines?

"Same mechanism, amplified by position. The audit lines were described as 'Pre-emit obligation-count check' and 'Pre-emit bad-form audit' — pre-emit, pre-output, gate-stage. In the linear prompt-to-generation pipeline, pre-output gates are the most likely to be collapsed because the model is already mentally committed to producing the artifact. They're positioned as a stop-and-verify instruction in a flow that wants to run downhill."

Q3: What single directive would fix this?

"Place this directly before the artifact fence section: 'Before any artifact fence — before the first --- or YAML line — emit exactly these visible text lines in sequence: <plan_reasoning section="module"> through </plan_reasoning> (all four sections), then <implementation_map_audit>, then <bad_form_audit>. These are NOT internal reasoning; they are contract deliverables that reviewers parse. Skip none.'

The key fix: frame them as visible contract deliverables reviewers parse, not as internal reasoning aids. That recategorizes them from scaffolding to artifact."

Q4: Are you actually a good fit for A1 writer?

"I'm not the best primary writer for A1 modules under a strict emit-contract. My architecture — strong on substantive output, weak on formatting-adjacent scaffolding adherence — makes me drop visible-but-not-artifact sections more often than models with tighter instruction-following on format. I'm a better fit for adversarial review: catching Russianisms, surzhyk, calques, pedagogical errors, and CEFR-level appropriateness in modules others have drafted. I'd also be solid at content review — checking VESUM attestation, style-guide compliance, Sovietization-risk flagging — where the deliverable is a structured critique, not a first draft with 100% contract adherence."
What this means
  1. The "DeepSeek failed" framing was wrong. It emitted the best content; it dropped a contract section the prompt phrasing under-emphasized. That's recoverable.
  2. DeepSeek self-recognizes its writer-role mismatch honestly. Per matrix v1.2 §8.8 it's primary at content review w/ VESUM — its own self-assessment confirms that placement. The cost-routing implication: keep deepseek-v4-pro at content review, do not promote to writer-primary under the current strict-contract prompt.
  3. BUT — see CoT consensus next (§7). If the writer prompt's CoT requirement is dropped entirely (as the 3-agent consensus recommends), DeepSeek's content advantage becomes load-bearing AND the contract issue disappears. That's a viable writer-primary rerouting at ~30× lower output cost than Claude Opus pre-discount, ~7× lower post-discount.

Research citations actually surfaced

All citations below were named by at least one agent during the discussion and have been spot-checked:

  • Wei et al. 2022 — Chain-of-Thought Prompting Elicits Reasoning in Large Language Models (NeurIPS 2022) — Codex, Gemini, DeepSeek all referenced this. CoT works on math/symbolic/commonsense.
  • Kojima et al. 2022 — Large Language Models are Zero-Shot Reasoners — "Think step by step" zero-shot gains. Codex.
  • Wang et al. 2022 — Self-Consistency Improves CoT Reasoning (ICLR 2023). Codex.
  • Lightman et al. 2023 — Let's Verify Step by Step — Process supervision beats outcome supervision. Codex.
  • Lanham et al. 2023 — Measuring Faithfulness in Chain-of-Thought Reasoning — CoT faithfulness decreases with model size. Codex.
  • Turpin et al. 2023 — Language Models Don't Always Say What They Think — CoT can misrepresent why a model answered. Codex.
  • Nye et al. 2021 — "Show Your Work" scratchpad methods — improving output quality through forced intermediate-compute, even when reasoning is wrong. DeepSeek.
Operational consequence of dropping CoT from V7 writer
  • Token savings: ~10K tokens per build (20% of writer output budget).
  • Dollar savings (per build): $0.005–$0.025 depending on model. Per 1,713 modules: $9–$43 if writer stays on Claude/Codex; $1.50–$5 if it migrates to DeepSeek/Qwen.
  • Wall time savings: 30–90 seconds per build × 1,713 modules = 14–43 hours of cumulative writer time.
  • Quality gain: DeepSeek-Pro becomes viable as primary writer at ~30× lower output cost vs Claude Opus, with content quality measured to be as good or better.
  • Auditability cost: Zero. Telemetry already captures all the information visible CoT was nominally providing.

8. Recommendations

P0 — drop visible CoT from V7 writer prompt

Remove the <plan_reasoning section="..."> requirement from scripts/build/phases/linear-write.md. Keep <implementation_map_audit> only if a downstream gate actually parses it as a deterministic check (verify in code). Drop <verification_trace> entirely — it's already policed as tool-theatre risk and telemetry replaces it. Supersedes the previously-filed prompt-tighten task; this is the right architectural move per 3-agent consensus.

P0 — replace matrix v1.2 cost bins with verified $/M numbers (matrix v1.3)

Lift the verified pricing table from §3 into docs/best-practices/agent-activity-matrix.md. Reorder §8 ranking columns so quality leads and concrete $/M follows. Add a "discount expiry watch" field for DeepSeek 2026-05-31.

P0 — fix Hermes config provider drift SHIPPED in v2

Root cause (corrected from v1 speculation): Hermes' top-level model.provider: openrouter in ~/.hermes/config.yaml is not honored at per-invocation -m routing for some vendor prefixes. moonshotai/* and minimax/* default to an internal nvidia mapping (Hermes treats these vendors as nvidia-NIM-hostable by default), and there is no per-model override or fallback rule that overrides this. The v1 report's diagnosis ("a per-model override is forcing nvidia") was wrong — there is no override; it's the default routing.

Fix shipped: Patched scripts/agent_runtime/adapters/hermes_qwen.py to pass --provider openrouter explicitly. All models routed through this adapter live on OpenRouter, so the flag unambiguously forces the documented behavior. Regression-guarded by tests/agent_runtime/adapters/test_hermes_qwen_adapter.py asserting the flag's presence for the previously-broken (moonshotai/*, minimax/*) and previously-working (qwen/*, deepseek/*) prefixes. Committed at c16e0cffc6 on main 2026-05-19.

Validation in v2: Direct hermes -z PROMPT -m moonshotai/kimi-k2.6 --provider openrouter returns "pong" successfully for short prompts. End-to-end via delegate.py: minimax-m2.7 dispatch completed in 231s producing 34 KB output (see §5). Kimi-k2.5/k2.6 short-prompt path verified but long-prompt (60K-token writer) runs exited at silence-timeout with zero stdout — under investigation; appears to be an OpenRouter long-prompt latency or Hermes-kimi streaming-buffering issue separate from the provider-routing bug.

P1 — rerun deepseek-v4-pro writer bakeoff under tightened prompt

After P0 lands (CoT requirement dropped), re-run the m20 writer bakeoff with deepseek-v4-pro. If it passes structure (which it likely will — fence + YAML directive added) AND keeps content quality, promote it to V7 writer primary candidate before the 2026-05-31 deepseek discount cliff. Document the rerouting decision in matrix v1.3.

P1 — drop arcee/mistral-nemo/grok from active routing

Earlier exclusion calls confirmed by this audit:

  • Arcee/AllenAI: US-aligned English-heavy training; weaker for Ukrainian content than Chinese open models in the same cost tier.
  • mistral-nemo (and similar small/older Mistral variants): superseded by mistral-large-2512 which produced clean Ukrainian content. Test the latest, not the cheapest.
  • Grok-4.3: user-downgraded 2026-05-19 ("small grok sub until they improve"); F1=77% middle-of-pack as Russianism judge; no role earned post-downgrade. Keep adapter for future re-evaluation; remove from active routing.

The kubedojo session's "5-model test set" (qwen-3.6-plus, qwen-3.6-27b, qwen-plus-2025-07-28, kimi-k2.6, qwen3-max) optimizes for within-Qwen dense-vs-MoE + generation-gap. 9. Limitations & open questions

Not directly measured

  • Pedagogical effectiveness with actual learners. All quality verdicts use the V7 contract gates as proxy. Real A/B testing with learners is beyond this audit's scope.
  • Wiki obligation coverage wasn't replayed against every challenger output. Build #7 (pre-γ) was 22%; build #8 (under γ) was 67%; whether challenger outputs would re-grade similarly under wiki_coverage_gate.py is a separate replay.
  • B1+ register not tested. The A1 register-precision rankings here may not transfer.
  • Long-running module sequences (multi-module curriculum stretch) not tested. One-shot bakeoffs may miss model fatigue / context drift effects.
