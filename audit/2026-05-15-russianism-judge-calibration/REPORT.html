Per-judge characteristic profile

claude-opus-4-7 — "Thorough proofreader"

  • Catches 7/7 dirty cases as dirty (perfect case-level recall)
  • Catches 5/5 clean cases as clean (perfect case-level precision)
  • Phrase-level recall 94% — missed only 1 of 16 gold flags (the preposition на завтра на 09:00 → о 09:00)
  • Phrase-level precision 79% — 3 phrase FPs all at sev=1 (defensible style points like обговорити терміни → строки, в → у euphony rule)
  • Tendency: flags real issues AND adds defensible sev=1 stylistic notes. For a curriculum review pipeline, this is exactly the desired behavior — better to over-flag with sev=1 (writer can dismiss) than miss a real calque.

gpt-5.5 — "Conservative validator"

  • Catches 5/8 dirty cases as dirty + 5/5 clean cases as clean
  • 2 case-level FNs: missed cal_dirty_meetup and cal_debatable_next_steps entirely (judged clean)
  • Phrase-level recall 69% — missed 4 of 13 gold flags (sev1-tolerant): залишу коментарі (sev=3), в будь-який момент (sev=2), на прийом (sev=2), не виходить (sev=2)
  • Phrase-level precision 90% — 1 FP, and it's a soft-FP (span mismatch on a real Russianism)
  • Tendency: very few false positives, but misses medium-severity calques. Conservatism is a feature for a validator role, a bug for a primary reviewer.

gemini-3.1-pro-preview — "Thorough but greeting-blind"

  • Catches 7/7 dirty cases as dirty (perfect case-level recall, ties opus-4-7)
  • Catches 4/5 clean cases as clean — single case-level FP is cal_clean_greeting:
    • Доброго дня! → Добрий день! (sev=2) — VESUM accepts the genitive, Antonenko has no entry. Predicted FP confirmed.
    • On the same cell: also flagged Як ваші справи? → Як ся маєте? (sev=3) — register preference, not a Russianism.
    • Combined: 2 spurious flags on a single clean case. If the curriculum content contains common greetings (and it will), this judge will fire on every one.
  • Phrase-level recall 87% — missed 2 sev≥2 gold flags
  • Phrase-level precision 81% — 1 additional phrase FP at sev=2 on debatable терміни → строки
  • Latency instability — judge calls regularly exceed 240s under load; required 480s timeout to complete the calibration set.
  • Tendency: high recall, but (a) systematic greeting FP fires on every common Ukrainian greeting, (b) latency unpredictable.

Findings synthesis

  1. gpt-5.5 wins on ensemble mean (2.59%). #1 or #2 under each individual judge. Frontier-cleanest Ukrainian producer in this 5-prompt suite.
  2. claude-opus-4-7 takes #1 under gpt-5.5 judge (1.44%) but drops to #3-4 under gemini and opus judges. Net #2 on ensemble mean.
  3. opus-4-7 as judge is systemically stricter — every model gets 2-4pp higher rate under opus-4-7 than under gemini/gpt judges. The judge's high-recall behavior pushes mean rates up uniformly without changing the bottom of the leaderboard.
  4. Bottom of the leaderboard is rock-solid. claude-haiku-4-5 is rank 10 under all 3 judges (spread = 0). Within-family scaling (smaller model = more calques) is universal: Haiku 2.5× Opus, gpt-mini 2.5× gpt-5.5, gem-flash 1.2-1.5× gem-pro.
  5. The frontier band (ranks 2-7) is judge-fragile. Spreads of 3-5 are common; the gap between #2 and #7 is only ~2.6 percentage points on ensemble mean. Headline rankings inside the band are NOT separable with single-judge eval.

Recommendation for the curriculum review position

Adopt claude-opus-4-7 as the primary reviewer. Configure the review pipeline so:

  1. Primary call: claude-opus-4-7 with the universal-judge prompt (retrieve Antonenko entries + scoring rubric). At sev≥2 the flag is escalated; at sev=1 the flag is logged but does not block publication.
  2. Optional second-opinion validator: when claude-opus-4-7 flags a sev≥2 issue, re-call gpt-5.5 with the same prompt. Both flag = block / require correction. opus-only sev≥2 = warn but allow (likely correct but worth human review). gpt-only flag should be rare given gpt-5.5's conservatism; treat as block.
  3. Calibration regression: run the 12-case calibration set monthly as a regression check on whichever judge is in production. If F1 falls below 80% on the same calibration, escalate.
  4. Do NOT use gemini-3.1-pro-preview as a primary reviewer. Its over-flagging of greeting genitives and register preferences will produce noise the writer team will learn to ignore — which destroys the value of the entire review pipeline.

Cost note

  • gpt-5.5 (ask-codex): ~20-25s/cell typical → $0.02-0.05/cell
  • claude-opus-4-7 (ask-claude): ~30s/cell typical → $0.05-0.08/cell
  • gemini-3.1-pro-preview: 25-300+ s/cell with frequent timeouts → unbudgetable

For curriculum-scale review (~6000 module activities × per-section audits), the claude-opus-4-7 + gpt-5.5 second-opinion configuration costs roughly $300-500/run on full corpus. Well within review budget on the Anthropic promo (+50% until 2026-07-13).

Methodology limits

  • Calibration set is n=12 + single-rater. Gold reflects one orchestrator's Antonenko-grounded reading, not a panel of native-speaker linguists. Treat as directional signal, not publishable claim.
  • gemini-3.1-pro latency is environmental. Yesterday it judged 55 cells fine; today it timed out on calibration cells with identical prompt size. Reproducible only on a quiet Gemini account.
  • Phrase-match is normalized-string equality. Different judges may surface the same calque on slightly different spans — counted as separate findings.
  • sev1-tolerant scoring. Debatable register calls don't penalize judge recall. Strict scoring (--strict-sev1) lowers recall numbers but doesn't change the ranking.
  • Single sample per (prompt, model) cell. Variance test on one cell showed ±2-3pp rate spread under n=5 retake. Ranks within ±1pp not separable.
  • gemini-3-pro-preview was attempted as 4th judge but DNF'd entirely. Unsuitable as production judge at any scale until upstream latency stabilizes.
