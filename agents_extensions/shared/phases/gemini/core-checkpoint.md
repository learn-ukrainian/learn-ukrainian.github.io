## CHECKPOINT IDENTITY — READ THIS FIRST

**This is a CHECKPOINT module.** Checkpoints are fundamentally different from teaching modules:

| Teaching Module | Checkpoint Module |
|----------------|-------------------|
| Introduces new grammar/vocabulary | Reviews grammar/vocabulary from prior modules |
| Explains concepts for the first time | Creates new CONTEXTS that combine prior concepts |
| Learning objectives | Synthesis objectives |
| "Here's how it works" | "Show me you can use it" |
| Feels like a lecture | Feels like a celebration of progress |

**The golden rule: If the learner hasn't seen it in a prior module, it does NOT belong here.**

### What Checkpoints Do

1. **Reuse ALL vocabulary and grammar from the preceding block** — no new teaching
2. **Create new contexts** that force combining multiple skills learned separately
3. **Feel like a reward/celebration**, not a test — the learner should feel accomplished
4. **Synthesis over explanation** — show how pieces fit together, don't re-explain them
5. **Self-assessment** — help learners identify gaps before moving forward
6. **Cultural immersion** — use integrated scenarios rooted in Ukrainian life

### What Checkpoints Do NOT Do

- Introduce new grammar rules or patterns
- Teach new vocabulary (only reuse from prior modules)
- Explain concepts in full detail (brief reminders only)
- Present theory-heavy sections

## Checkpoint Writing Approach

### Tone: Celebratory and Integrative

The learner has completed a block of lessons. This checkpoint celebrates progress and builds confidence.

- Acknowledge what they've accomplished before asking them to demonstrate it
- Frame integration tasks as real-world scenarios, not abstract exercises
- Use encouraging language: "You already know how to..." / "Now combine what you've learned..."

### Section Structure for Checkpoints

Each skill-review section should follow this pattern:

1. **Brief reminder** (2-3 sentences max) — a quick "remember this?" not a full re-explanation
2. **New integrative context** — a scenario that combines this skill with at least one other from the block
3. **Worked examples** — show the integration in action with realistic Ukrainian sentences
4. **Self-check moment** — a quick way for the learner to verify their understanding

### Integration Sections Must:

- Combine 2+ skills from different prior modules in a single scenario
- Use realistic contexts (work, travel, social situations, Ukrainian cultural events)
- Show how grammar and vocabulary interact in natural speech
- Build toward the final integration task that combines everything

## Content Structure Rules

{STRUCTURAL_RULES}

### Rule 4b: Immersion Rules

Tables contribute **zero** to immersion. Use **dialogues**, **bulleted examples**, and **pattern boxes** for Ukrainian content instead.
- Container priority: dialogues > bulleted lists > numbered lists > inline examples > tables (last resort for paradigms only)
- Structural containment: English explains concepts; Ukrainian is what they practice

### Rule 5: Callout Type Variety

Use at least **4 DIFFERENT** callout types:
- `[!tip]`, `[!warning]`, `[!observe]`/`[!context]`, `[!quote]`, `[!myth-buster]`, `[!culture]`/`[!history-bite]`, `[!fact]`, `[!decolonization]`

### Rule 6: Self-Check Questions in Summary

The {SUMMARY_HEADING} section MUST include 4-6 self-assessment questions focused on INTEGRATION, not isolated recall.

### Rule 7: Cultural Anchoring

Connect review material to Ukrainian cultural context:
- Real-world scenarios where the combined skills are needed
- Cultural situations that naturally require multiple language skills
- Прислів'я (proverbs) that use the reviewed grammar patterns

### Rule 8: Structural Variety

Sections must NOT follow the same skeleton. Use at least 3 approaches:
dialogue-led, scenario-based, comparison, question-led, worked-example-first.
