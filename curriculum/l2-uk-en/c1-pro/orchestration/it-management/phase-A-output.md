## Мовні норми та стандарти
- **ДСТУ ISO 21502:2022**: Визначає термінологію та процеси управління проєктами, програмами та портфелями. Важливо використовувати термін «врядування» (governance) замість «управління» у контексті стратегічного нагляду.
- **ДСТУ 4163:2020**: Встановлює правила оформлення реквізитів у документах (назва організації, дата, підпис, гриф затвердження), що необхідно для офіційної ІТ-документації.
- **Новий правопис (2019)**: Закріплює написання слова «проєкт» через «є», що є критичним для професійної грамотності.

## Section-Mapped Research Notes

### Розминка — ІТ як стратегічна функція
Ключова роль ІТ-менеджера (CIO/CTO) у цифровій трансформації. Статистика українського ІТ-ринку (частка у ВВП, кількість спеціалістів). Баланс між Hard Skills (архітектура, інфраструктура) та Soft Skills (лідерство, стратегічне планування).

### Методології управління проєктами
Порівняння Waterfall (каскадна модель) та Agile (гнучка розробка). Стандарти ДСТУ ISO 21502 як теоретична база. Термінологія Scrum: артефакти (backlog, increment), події (sprint planning, daily scrum). Поняття WIP (Work in Progress) у Kanban та методи візуалізації потоку.

### DevOps та операційна діяльність
Філософія «You build it, you run it». Автоматизація через CI/CD (Continuous Integration / Continuous Deployment). Метрики надійності: SLA (Угода), SLO (Цілі), SLI (Показники). Відмінність між моніторингом (що сталося) та обсервабільністю (чому це сталося).

### Управління ІТ-командою
Цикл життя співробітника: наймання, адаптація (onboarding), розвиток, звільнення. Інструменти оцінки: 360-degree feedback, OKR (Objectives and Key Results). Специфіка управління розподіленими командами (синхронна vs асинхронна комунікація). Профілактика професійного вигорання в умовах високого темпу розробки.

### Зразки ІТ-документації
Структура Статуту проєкту (Project Charter) згідно з PMI/ISO. Формат звіту про завершення спринту (Sprint Review). Правила ділового листування в ІТ згідно з ДСТУ 4163:2020. Типові фрази для постановки завдань у Jira/Azure DevOps.

### Практика — ІТ-менеджмент
Підготовка до Kickoff meeting. Структура презентації: візія проєкту, обсяг робіт (scope), часові рамки (timeline), матриця відповідальності (RACI).
