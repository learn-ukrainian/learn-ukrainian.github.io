# Ohoiko corpus intake — source inventory summary

This is the committed, reviewable stand-in for `ohoiko-corpus-intake.json` — the full
flat source inventory (111,200 rows, ~31 MB). That JSON is **deterministic, regenerable
output** of committed code run over the private Ohoiko corpus, so it is **gitignored, not
committed** (orchestrator decision, #4223): committing 31 MB of regenerable output would
bloat git history permanently. The durable review-decision record is the per-batch ledger
under `../source-inventory-review-decisions/2026-07-14-ohoiko-corpus-intake-batch-*.yaml`
(21 batches, 20,848 rows). This file preserves the counts and a content hash so a
regenerated inventory can be verified byte-for-byte.

## Regeneration command (main checkout only)

The private corpus lives under `docs/references/private/` in the **main checkout** and is
itself gitignored (copyright, #4963), so this must be run there — never in a sparse
subagent worktree, which excludes `docs/references/private/` and `data/`.

```bash
# From the main checkout root. Pass --private-root as an ABSOLUTE path so the run is
# independent of the current working directory.
.venv/bin/python -m scripts.lexicon.ohoiko_atlas_intake \
  --private-root /Users/krisztiankoos/projects/learn-ukrainian/docs/references/private \
  --inventory-out data/lexicon/source-inventory/ohoiko-corpus-intake.json
```

`--inventory-out` is required to write the JSON to this committed-inventory path; the
script's default `--inventory-out` is `/tmp/atlas-ohoiko-corpus-inventory.json`. The
`--inventory-path` string embedded in the ledger rows
(`data/lexicon/source-inventory/ohoiko-corpus-intake.json`) is already the default, so it
need not be passed.

To also re-emit the review ledger (writes a single monolithic YAML; re-split into batches
with the same 1,000-row chunking used here):

```bash
.venv/bin/python -m scripts.lexicon.ohoiko_atlas_intake \
  --private-root /Users/krisztiankoos/projects/learn-ukrainian/docs/references/private \
  --inventory-out data/lexicon/source-inventory/ohoiko-corpus-intake.json \
  --ledger-out /tmp/ohoiko-corpus-intake-ledger.yaml \
  --batch-id ohoiko-corpus-intake-2026-07-14 \
  --batch-label "Ohoiko corpus Phase 2 extraction intake" \
  --reviewed-at 2026-07-14
```

## Ledger validation without the inventory present

`scripts/audit/source_inventory_review_decisions.py` validates the committed batch
ledgers. When the regenerable inventory JSON is absent (fresh clone / CI), each decision
row is still structurally validated — the `source_inventory.key` hash pins
`lemma + path + locator` — and the record-existence + `source_id`/`source_family`
cross-check is skipped (fail-open on the regenerable dependency). Regenerate the JSON to
this path to restore the full cross-check locally.
